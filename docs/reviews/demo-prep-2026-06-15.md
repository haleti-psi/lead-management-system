## 2. How to PROVE each requirement is satisfied (3 layers of evidence)

1. **Traceability** → the coverage report maps every FR → backend file → frontend file → test file (file:line). Show a few rows live.
2. **Tests** → run the suite; each FR has unit/component tests. Green tests = the logic does what the BRD says.
3. **Live behaviour** → for the user-facing flows, click through the actual screen (§3).

> For backend/system FRs with no UI (SLA engine, outbox, audit chain, LOS hand-off), **prove via tests + a code walk**, not a click-through — that's expected and fine.

## 4. To make it live-demoable (prerequisites)
The UI renders, but to actually log in you need: **API + PostgreSQL (`lms_dev`) + Redis running, and a seeded admin user** (there's no first-admin bootstrap yet — that's a gap to close before any live demo). Quickest path: `/local-deployment` (after `/project-config-init`) + seed one admin. *Without this, demo the live screens from a pre-recorded clip and lean on tests + traceability.*
