## Checklist Results

### ✓ 1. Every entity named in any BRD FR has a table — PASS
`information_schema` reports **46 base tables** = the 45 BRD §5.1 entities + the `orgs` infra seam. The §5.1 inventory (rows 1–45) maps 1:1 to §5.2 definitions and to `CREATE TABLE` statements (independently confirmed in the Gate-A re-audit). No BRD entity is missing a table.

### ✓ 2. Every workflow relationship has a FK — PASS
**194 foreign-key constraints** present. The BRD §5.3 relationships are realized as named `fk_<table>_<column>` constraints with explicit `ON DELETE` actions, including the previously-unrealized `ImportJob → Lead` (now `leads.import_job_id` → `import_jobs`). Genuinely polymorphic columns (`audit_logs.entity_id`, `event_outbox.aggregate_id`, `configuration_versions.config_ref`, `notification_preferences.subject_ref`) intentionally carry no FK and are documented (DATA_MODEL.md #4).

### ✓ 3. Every status/state field has an enum type — PASS
**69 enum types** created from the §5.5 catalog (single source of truth); enum value sets match §5.5 exactly. All status/stage/type columns use these native enums (verified clean load).

### ✓ 4. Every M:N relationship has a junction table — PASS
Confirmed present: `duplicate_matches` (Lead↔Lead), `role_permissions` (Role↔capability), `notification_preferences` (subject↔purpose↔channel). Each has the appropriate composite UNIQUE constraint.

### ✓ 5. Soft-delete requirements implemented — PASS
**4 tables carry `deleted_at`** (`leads`, `lead_identities`, `customer_profiles`, `documents`) for the retention/anonymise flow (FR-115); master/config tables use `is_active`. Matches BRD §4.3 soft-delete convention; partial index `WHERE deleted_at IS NULL` on `leads`.

### ✓ 6. Multi-tenancy consistent (org_id) — PASS
Query for business tables lacking `org_id` (excluding the `orgs` root) returns **0**. Every business table carries `org_id NOT NULL DEFAULT '…0001' REFERENCES orgs(id)` (the BRD §4.3 single-tenant seam).

### ✓ 7. Every table has created_at & updated_at TIMESTAMPTZ — PASS
**46/46 tables** have both `created_at` and `updated_at`; a type check returns **0** columns that are not `timestamp with time zone`. `set_updated_at` trigger applied to 43 tables (the 3 append-only tables — `consent_records`, `stage_history`, `audit_logs` — intentionally omit the trigger; documented).

### ✓ 8. Assumptions documented in DATA_MODEL.md — PASS
DATA_MODEL.md "Assumptions & Gaps" lists **14 items**, including every schema-vs-BRD delta (orgs seam, enum type-name renames, `leads.product_code`+`product_config_id`, polymorphic FKs, deferred audit FKs, `import_job_id` addition, `data_sharing_logs` RESTRICT) and the deferred decisions (migration channel, consent-bootstrap, audit-chain serialization, business calendar).

### ✓ 9. Table → source-FR metadata — PASS
**45 `-- source_fr:` comments** — one per BRD entity table (the `orgs` infra table is intentionally exempt). Each links the table to its originating FR(s).

### ✓ 10. schema.sql passes syntax validation — PASS
Loaded into a throwaway PostgreSQL 18 database with `psql -v ON_ERROR_STOP=1` → **exit 0**, twice (initial + after the quality fixes). The seed bootstrap committed (9 roles + system actor), proving the deferred-FK / atomic-write design works. All features used exist in the declared target (PG 15+). Also live in `lms_dev` for inspection.
