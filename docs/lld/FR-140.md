## Summary

FR-140 defines the **`IntegrationGateway`** — the single chokepoint that every module must use when calling an external provider. It is a **foundation FR** (build before M8/M9/M11 or any consumer module) and does not modify `leads` or call `LeadService`.

The five responsibilities of FR-140:

1. **Idempotency:** every outbound call carries an `Idempotency-Key`; Redis (`idem:gw:{key}`) plus a partial unique index on `integration_logs.idempotency_key` (`uq_integration_idempotency`) prevent duplicate provider calls and return the original result on replay (`IDEMPOTENT_REPLAY` detail reason, HTTP 200).
2. **Retry / exponential backoff:** on provider failure the gateway updates the `integration_logs` row to `retrying` and enqueues a Cloud Tasks retry task (max 3 attempts, base 1 s × 2^n, ±20% jitter). Exhausted retries transition the log to `failed` and enqueue a dead-letter task.
3. **Circuit breaker:** Redis key `cb:{integrationKind}` tracks consecutive failures; once the threshold (default 5) is reached the circuit opens for `CIRCUIT_OPEN_TTL` seconds (default 60 s). While open, calls fail fast with `UPSTREAM_UNAVAILABLE` (503) without hitting the provider.
4. **Observability:** every outbound call writes an `integration_logs` row (pending → success/failed/retrying); the integration monitor endpoint surfaces these rows with filtering and pagination.
5. **Webhook subscription CRUD:** IT/ADMIN registers outbound webhook targets per `event_code`; the delivery worker (FR-141 boundary) reads `webhook_subscriptions` rows. Inbound LOS status webhooks (FR-082) are verified by HMAC before processing.

## Auth Check

All three FR-140 endpoints share the same guard chain:

```
JwtAuthGuard (global)
  — validates access token from Authorization header or httpOnly cookie
  — 401 AUTH_REQUIRED if absent, expired, or malformed

AbacGuard + @Requires('configuration', () => ({ scope: 'A' }))
  -> EntitlementService.can(user, 'configuration', { orgId })
  -> Permitted: ADMIN (scope A), HEAD (scope A)
  -> Denied: RM, BM, SM, KYC, DPO, PARTNER, CUSTOMER -> 403 FORBIDDEN
```

Notes per `auth-matrix.json`:
- `configuration` is held by ADMIN (A), HEAD (A), BM (B), KYC (B), DPO (B). The `scope: 'A'` requirement excludes BM/KYC/DPO (branch-scoped only). HEAD holds scope A but the BRD FR-140 "Monitor = IT/ADMIN" guidance is primary intent; the guard should enforce scope A and let both ADMIN and HEAD through.
- `@Public()` is NOT applied to any FR-140 endpoint.
- The inbound LOS status webhook (`POST /api/v1/los/webhooks/status`) is `service_to_service_only` and uses `LosWebhookGuard` (HMAC); it is NOT a JWT endpoint and is documented as part of FR-082's flow. FR-140 implements the HMAC guard infrastructure reused by FR-082.

### Inbound HMAC verification (LosWebhookGuard — `guards/los-webhook.guard.ts`)

Implemented as a Nest guard; applied to `POST /api/v1/los/webhooks/status` in FR-082's controller. Documented here as part of FR-140's IntegrationGateway infrastructure:

```typescript
import { createHmac, timingSafeEqual } from 'crypto';

// key from LOS_WEBHOOK_HMAC_SECRET loaded via ConfigService (Secret Manager)
const sig  = request.headers['x-los-signature'];   // SHA-256 HMAC hex sent by LOS
const body = rawBodyBuffer;                         // captured before JSON parse (NestJS RawBody)
const expected = createHmac('sha256', secret)
  .update(body)
  .digest('hex');

if (
  typeof sig !== 'string' ||
  sig.length !== expected.length ||
  !timingSafeEqual(Buffer.from(sig, 'hex'), Buffer.from(expected, 'hex'))
) {
  throw new ForbiddenException({ code: 'FORBIDDEN', message: 'Webhook signature mismatch.' });
}
```

`timingSafeEqual` prevents timing-oracle attacks. The guard rejects bad signatures with 403 FORBIDDEN; no further processing occurs; the rejection is logged (without the raw body).

## Backend Flow

### GET /api/v1/admin/integrations

1. **CorrelationMiddleware** — attaches or generates `X-Correlation-Id`; sets it on response.
2. **JwtAuthGuard** — validates JWT; loads user + ABAC attributes (role, orgId, scope) onto request context; returns 401 on failure.
3. **AbacGuard** — calls `EntitlementService.can(user, 'configuration', { orgId })` with scope resolver returning `A`; returns 403 FORBIDDEN if denied.
4. **ValidationPipe** — parses `IntegrationMonitorQueryDto` via Zod; returns 400 VALIDATION_ERROR on failure.
5. `IntegrationController.listLogs(query)` delegates to `IntegrationService.listLogs(query, user)`.
6. Service builds `filters` map from validated query params; calls `IntegrationRepository.listLogs(filters, page, limit, sort)`.
7. Repository executes parameterised Kysely query (no string interpolation); `LIMIT` is always set (max 100).
8. **ResponseEnvelopeInterceptor** wraps result: `{ data: rows, meta: { correlation_id, pagination: { page, limit, total } }, error: null }`.

### GET /api/v1/admin/webhooks

Steps 1–4 identical (capability `configuration`, scope A resolver).
5. `IntegrationService.listWebhooks(page, limit)`.
6. `WebhookRepository.listWebhooks(page, limit)` — selects all columns **except** `secret_ref`.
7. ResponseEnvelopeInterceptor wraps result.

### POST /api/v1/admin/webhooks

Steps 1–3 identical (JwtAuthGuard, AbacGuard — `configuration`, scope A).
4. **ValidationPipe** validates `CreateWebhookDto`; 400 on failure.
5. `IntegrationService.createWebhook(dto, idempotencyKey, user)`:

   **a. Idempotency gate (Redis):**
   - If `Idempotency-Key` header is present:
     - Check `HGETALL idem:webhook:{key}` in Redis.
     - Hit with `status=success` → return cached response body; set `error.detail.reason=IDEMPOTENT_REPLAY`; return HTTP 200. No DB write.
     - Hit with `status=in_flight` → return 409 CONFLICT (concurrent duplicate in-flight).
     - Miss → `HSET idem:webhook:{key} status in_flight` with TTL 24 h.

   **b. `UnitOfWork.run(async (tx) => { … })`:**
   - `WebhookRepository.create(dto, currentUser, tx)` → `INSERT INTO webhook_subscriptions`.
   - `AuditAppender.append({ action: 'config_change', actorId: user.userId, targetId: newRow.webhook_subscription_id, orgId }, tx)` — writes intent to audit queue within the transaction.

   **c. Post-commit:**
   - If `Idempotency-Key` was present: `HSET idem:webhook:{key} status success` + cache serialised response body; TTL 24 h.
   - Return HTTP 201 with created row (excluding `secret_ref`).

   **d. On any error:**
   - `UnitOfWork` rolls back (no partial row, no orphaned audit entry).
   - If `Idempotency-Key` was set to `in_flight`: update to `failed`.
   - Rethrow — exception filter maps to `INTERNAL_ERROR` (500).

### IntegrationGateway.call() — Core Infrastructure (called by all external-facing modules)

This is not an HTTP handler but the central piece of FR-140:

```
IntegrationGateway.call<T>(port: Port<T>, request: IntegrationRequest, options: GatewayOptions): Promise<T>
```

1. **Validate idempotency key** format if `options.idempotencyKey` is provided (IdempotencyKeySchema). Throw `VALIDATION_ERROR` if malformed.

2. **Redis idempotency check** (`idem:gw:{idempotencyKey}`):
   - `success` → return deserialised cached result (mark as idempotent replay in returned object).
   - `in_flight` → throw `CONFLICT` (409) — concurrent duplicate in progress.
   - Miss → set `in_flight`, TTL 24 h.

3. **Circuit breaker check** (Redis key `cb:{integrationKind}`):
   - State `open` and `opens_at` > `now()` → throw `UPSTREAM_UNAVAILABLE` (503) immediately; write `integration_logs` row with `status=failed`, `error_code='CB_OPEN'`; skip adapter call.
   - State `half_open` or not present → proceed.

4. **Write `integration_logs` row** (`status=pending`) via `IntegrationLogRepository.createLog()`. This is a standalone DB write outside the calling module's UoW (the gateway must not hold the caller's DB connection open for the duration of the external call).

5. **Dispatch to port adapter** using native `fetch` with `AbortController` timeout (`REQUEST_TIMEOUT_MS`, default 10 000 ms). Port interface: `port.call(request): Promise<ProviderResponse>`.

6. **On success (2xx):**
   - Update `integration_logs` → `status=success`, `http_status`, `completed_at=now()`.
   - Reset circuit breaker: `DEL cb:{integrationKind}:failures` or reset counter.
   - If idempotency key: `HSET idem:gw:{key} status success` + cache serialised result, TTL 24 h.
   - Return provider response to caller.

7. **On failure (provider 4xx/5xx/timeout/network error):**
   - Update `integration_logs` → `status=failed`, `error_code`, `retry_count` incremented.
   - Increment circuit breaker failure counter: `INCR cb:{integrationKind}:failures` (TTL 60 s window). If `>= CIRCUIT_BREAKER_THRESHOLD` (default 5): `SET cb:{integrationKind} open`, `HSET cb:{integrationKind} opens_at <timestamp + CIRCUIT_OPEN_TTL>`.
   - If `retryCount < MAX_RETRIES` (3):
     - Enqueue Cloud Tasks retry task with exponential backoff (`2^retryCount * 1000 ms base`, ±20% jitter).
     - Update `integration_logs` → `status=retrying`.
   - If `retryCount >= MAX_RETRIES`:
     - Update `integration_logs` → `status=failed` (final).
     - Enqueue dead-letter queue task for operational review.
   - If idempotency key: `HSET idem:gw:{key} status failed`, TTL 24 h.
   - Throw `UPSTREAM_UNAVAILABLE` (503) to the caller.

## State Machine + Transaction Boundaries

### IntegrationLog — `integration_status`

Per `docs/contracts/state-machines.md §IntegrationLog`:

```
pending --> success
pending --> failed --> retrying --> success
                              --> failed (max retries / dead-letter)
```

All transitions are system-managed by `IntegrationGateway` only. No user-initiated transitions exist.

| Transition | Trigger | Writer | Side effects |
|---|---|---|---|
| `pending → success` | Provider 2xx | `IntegrationGateway` (UPDATE) | Idempotency cache updated to `success`; circuit counter reset |
| `pending → failed` | Provider error / timeout | `IntegrationGateway` (UPDATE) | Circuit counter incremented; Cloud Tasks retry task enqueued (if retries remain) |
| `failed → retrying` | Retry task accepted by Cloud Tasks | `IntegrationGateway` (UPDATE) | `retry_count++`; backoff delay applied by Cloud Tasks |
| `retrying → success` | Retry 2xx | `IntegrationGateway` (UPDATE) | Same as `pending → success` |
| `retrying → failed` | Max retries exhausted | `IntegrationGateway` (UPDATE) | Dead-letter queue task; Cloud Monitoring alert (`UPSTREAM_UNAVAILABLE` + `INTERNAL_ERROR` class) |

**Transaction boundaries for POST /admin/webhooks:**

```
UnitOfWork.run(tx):
  INSERT webhook_subscriptions              [WebhookRepository — within tx]
  AuditAppender.append(webhook_created, tx)   [audit intent written to queue within tx]
COMMIT
-- Post-commit side effects (not in the tx; retryable if they fail):
  Redis idempotency key -> status=success + cache serialised row
```

`IntegrationGateway.call()` intentionally does **not** participate in the calling module's UoW. The `integration_logs` INSERT and UPDATE are short-lived standalone DB writes so the caller's connection is never held open across the network round-trip to the external provider.

## External Service Calls via IntegrationGateway

FR-140 defines the gateway infrastructure. All provider ports are defined here; calling modules never instantiate adapters directly.

| Port interface | Prod adapter | Dev/test adapter | Env vars |
|---|---|---|---|
| `LosPort` | `LosHttpAdapter` | `LosMockAdapter` (`LOS_MOCK=true`) | `LOS_BASE_URL`, `LOS_API_KEY`, `LOS_WEBHOOK_HMAC_SECRET`, `LOS_MOCK` |
| `KycPort` | `PanAdapter`, `CkycAdapter`, `DigiLockerAdapter`, `AadhaarAdapter`, `VcipAdapter` | mock adapters | `PAN_PROVIDER_URL/KEY`, `CKYC_URL/KEY`, `DIGILOCKER_*`, `AADHAAR_*` |
| `NotificationChannelPort` | `SmsAdapter`, `WhatsappAdapter`, `EmailAdapter` | `MockChannelAdapter` | `SMS_PROVIDER_URL/KEY/SENDER_ID`, `WHATSAPP_BSP_URL/KEY/WABA_ID`, `EMAIL_PROVIDER_URL/KEY/FROM`, `TRAI_DLT_ENTITY_ID` |
| `TelephonyPort` | `CtiAdapter` (Phase 1.5) | mock | `CTI_*` |
| `AaPort`, `GstPort`, `AssetVerificationPort` | real (Phase 1.5) | mock | `AA_*`, `GST_*`, `VAHAN_*` |

All adapters use native `fetch` (no axios — see `dependency-register.md`). `AbortController` enforces `REQUEST_TIMEOUT_MS` per call.

**Common failure handling (all ports):**
1. Provider error / timeout → `UPSTREAM_UNAVAILABLE` (503) + `integration_logs(status=failed/retrying)` + Cloud Tasks retry (exponential backoff, max 3).
2. Circuit breaker opens on `CIRCUIT_BREAKER_THRESHOLD` (default 5) consecutive failures → fast-fail with 503.
3. Poison message (max retries exceeded) → dead-letter queue → operational alert.
4. Idempotency key deduplicates; replays return the original result without re-calling the provider.

## UI Component Tree

The Integration Monitor and Webhook management surfaces are ADMIN-only pages. All components reuse existing shared primitives (no new primitives created).

```
AdminIntegrationsPage
  (apps/web/src/app/admin/integrations/AdminIntegrationsPage.tsx)
├── AppShell                        (role-filtered nav; ADMIN/HEAD role shows this route)
│   └── PageHeader title="Integration Monitor"
├── IntegrationMonitorFilters
│   (apps/web/src/components/integration/IntegrationMonitorFilters.tsx)
│   ├── Select (shadcn)   filter[integration] — IntegrationKind enum options
│   ├── Select (shadcn)   filter[status]      — IntegrationStatus enum options
│   ├── Select (shadcn)   filter[direction]   — outbound | inbound
│   └── Button "Clear filters"
├── DataTable (shared — server pagination, column visibility)
│   columns:
│     Integration (display label from IntegrationKind)
│     Direction (badge: outbound/inbound)
│     Lead ID (link to lead detail if present)
│     Correlation ID
│     Status (StatusChip: pending/success/failed/retrying)
│     HTTP Status
│     Retry Count
│     Completed At (IST, dd-MM-yyyy HH:mm)
│     Created At (IST)
│   pagination: page/limit via URL search params; default limit 25; max 100
├── EmptyState (no logs match filters)
└── ErrorState (API error — FORBIDDEN / INTERNAL_ERROR)

AdminWebhooksPage
  (apps/web/src/app/admin/webhooks/AdminWebhooksPage.tsx)
├── AppShell
│   └── PageHeader title="Webhook Subscriptions" actions=[Button "Add Webhook"]
├── DataTable (shared)
│   columns:
│     Event Code (StatusChip variant="neutral")
│     Target URL
│     Active (boolean badge)
│     Last Status (StatusChip: queued/sent/delivered/failed)
│     Created At (IST)
│   pagination: page/limit; default 25
├── CreateWebhookDrawer
│   (apps/web/src/components/integration/CreateWebhookDrawer.tsx)
│   └── EntityForm (react-hook-form + Zod resolver)
│       ├── Select (shadcn)  — eventCode (EventCode enum)
│       ├── Input            — targetUrl  (inline error: "Must begin with https://")
│       ├── Input            — secretRef  (inline error: "Must be a valid Secret Manager path")
│       └── Button "Create" (submit; disabled while mutation in-flight)
│           on success: Toast "Webhook created"; Drawer closes;
│                       invalidate useQuery(['admin','webhooks'])
│           on VALIDATION_ERROR: EntityForm maps error.fields to inline field errors
├── EmptyState
└── ErrorState
```

**State management:** TanStack Query.
- `useQuery(['admin', 'integrations', filters, page, limit])` — stale-time 30 s; refetch on focus.
- `useQuery(['admin', 'webhooks', page, limit])` — stale-time 60 s.
- `useMutation` for `createWebhook` (POST /admin/webhooks with optional `Idempotency-Key`).

**Accessibility:** all interactive elements keyboard-reachable; Select and Drawer use Radix primitives (built-in ARIA); WCAG 2.1 AA; `prefers-reduced-motion` respected by Radix.

## Shared Utilities to Use

| Utility | Location | Usage in FR-140 |
|---|---|---|
| `UnitOfWork.run(fn(tx))` | `core/db/` | Webhook creation — wraps INSERT + AuditAppender call |
| `Db` (Kysely instance) | `core/db/` | All repository queries |
| `AuditAppender` | `core/audit/` | Emit `webhook_subscription_created` event on POST /webhooks |
| `EntitlementService.can(user, capability, resource)` | `core/auth/` | ABAC check (`configuration`, scope A) in AbacGuard |
| `JwtAuthGuard` / `AbacGuard` / `@Requires` | `core/auth/` | Auth guard chain on all three endpoints |
| `CorrelationMiddleware` | `core/http/` | `X-Correlation-Id` propagated into `integration_logs.correlation_id` |
| `ResponseEnvelopeInterceptor` | `core/http/` | Uniform `{ data, meta, error }` envelope on all responses |
| `AllExceptionsFilter` | `core/http/` | Maps unhandled errors to `INTERNAL_ERROR` (500); no stack in response |
| `ioredis` client (shared) | `core/` | Idempotency cache (`idem:gw:*`, `idem:webhook:*`) + circuit breaker state (`cb:*`) |
| `@google-cloud/tasks` | `core/integration/` | Cloud Tasks retry queue enqueue + dead-letter |
| `PaginationParams` (Zod) | `packages/shared/types/` | `page` default 1, `limit` default 25 max 100 |
| `ApiEnvelope<T>` / `ApiError` | `packages/shared/types/` | Response type shapes |
| `@shared/enums` — `IntegrationKind`, `IntegrationStatus`, `IntegrationDirection`, `EventCode`, `DeliveryStatus` | `packages/shared/enums/` | Enum values — never redefined locally |
| `DataTable`, `AppShell`, `PageHeader`, `EmptyState`, `ErrorState`, `Toast`, `StatusChip`, `EntityForm`, `Drawer` | `apps/web/src/components/` | All UI primitives — reuse only |
| `apiClient` | `apps/web/src/lib/api/` | Typed fetch with envelope + correlation + auth cookie |
